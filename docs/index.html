STRAIDE:
A Research Platform for Shape-Changing Spatial Displays based on Actuated Strings

A project by the Interactive Media Lab Dresden:
Severin Engert, Konstantin Klamka, Andreas Peetz, Raimund Dachselt

We present STRAIDE, a string-actuated interactive display environment that allows to explore the promising potential of shape-changing interfaces for casual visualizations. At the core, we envision a platform that spatially levitates elements to create dynamic visual shapes in space. We conceptualize this type of tangible midair display and discuss its multifaceted design dimensions. Through a design exploration, we realize a physical research platform with adjustable parameters and modular components. For conveniently designing and implementing novel applications, we provide developer tools ranging from graphical emulators to in-situ augmented reality representations. To demonstrate STRAIDE's reconfigurability, we further introduce three representative physical setups as a basis for situated applications including ambient notifications, personal smart home controls, and entertainment. They serve as a technical validation, lay the foundations for a discussion with developers that provided valuable insights, and encourage ideas for future usage of this type of appealing interactive installation.

Construction

Firmware

Server

A NodeJS server acts as the central heart of any STRAIDE instance. It can be run on any computer connected to the same network as the Motor Towers, preferably a thin client like a RaspberryPi 4B. Install NodeJS and npm, then run npm install, and start node controller.js.

 Firmware>Server

Motor Controller

Four stepper motors are connected to a central CNC shield stacked onto an Arduino Uno. This microcontroller board runs a custom Arduino Sketch, evaluating incoming Ethernet messages, and positioning the motors accordingly.

 Firmware>Motor_Uno

Color Controller

Illuminated elements can be connected to a central PCB with a WiFi-capable microcontroller board, preferably an ESP-32. This runs a custom Arduino Sketch which connects to the central server, receives new color values and shows then using the NeoPixel library.

 Firmware>Color_ESP32

Manual

How to use?

This section provides an interactive view of the construction of one STRAIDE instance: 8x8 elements in a dense matrix arrangement with a custom frame.

It includes the following information:

  • Logical structure: STRAIDE is comprised of nested components, their structure is shown as a hierachical tree.
  • 3D Model: For each major component, a 3D view of the assembly is shown and can be interactively inspected.
  • BIM: For each individual components, a list of required parts is given. This needs to be multiplied by the occurence of individual component.
    • A sum of all required parts per component is calculated and shown.
  • Fabrication Files: For all parts that need to be individually fabricated, all files are provided.
    • This includes .svg (lasercutting), .stl (3D printing), .zip (Gerber files for PCBs).
  • Assembly Steps: For each component, a set of instruction is provided explaining the required assembly steps.
    • An overall Manual can be shown as well as downloaded.
If you really want to build an exact replica of our system, then you should follow all the provided steps. If this is not the case, you should identify required components based on what you want to achieve. Most often, this will result in the Actuation Unit>Motor Tower and parts of the Actuation Unit>Control Unit.
